One request. Four controlled stages.

The agent gets a persistent job ID immediately. Generation continues in a detached local worker, then returns only after every output passes media and hash checks.

  1. 1.0

    Describe

    Claude or Codex calls start_website_video with prompt, motion, ratio, and output path.

  2. 2.0

    Frame

    Use an existing image, try the logged-in ChatGPT browser, or fall back to Grok only before submission.

  3. 3.0

    Generate

    Grok Build submits image_to_video through the paid-member session with API-key auth disabled.

  4. 4.0

    Verify

    FFmpeg and ffprobe decode, normalize, hash, and publish the website bundle without overwriting existing files.

Routing is conservative on purpose.

Membership quota is not safely retryable. If a provider may already have received a request, the job stops and records an unknown submission state.

What “no API” means

No developer API, no developer API key, and no direct xAI or OpenAI REST adapter. Official membership clients still connect to their own services.

Uses

  • Logged-in Grok membership
  • Logged-in ChatGPT browser
  • Local MCP, SQLite, FFmpeg

Refuses

  • XAI_API_KEY
  • OPENAI_API_KEY
  • Public API fallback

Quota safety is a state machine.

The hard part is not starting a generation. It is proving when a retry is safe, keeping two agents from spending the same quota, and refusing corrupted output.

Read the Chinese architecture
Before launch
Known API-key variables are removed. ChatGPT is pinned to the web backend.
During generation
SQLite reserves output names, idempotency reuses jobs, and Grok media work is serialized.
After generation
Every file is decoded and hashed. The manifest publishes last and is rechecked on every completed read.

Precise answers, before you install.

Does this bypass membership limits?

No. It spends the normal paid-member quota through the logged-in official clients.

Is ChatGPT required for video?

No. Grok can generate the first frame when ChatGPT fails safely before submission, or you can supply an existing image.

What happens when submission status is unknown?

The job stops as submitted_unknown. It never retries automatically.

Which systems are verified?

macOS is verified with Codex CLI/Desktop, Claude Code, and Claude Desktop. Linux and Windows are not claimed yet.

Keep developer keys out of the media pipeline.
